🔧 实体编辑修复测试

🎯 修复内容

✅ 工作副本状态计算

添加了 calculateWorkingCopyStatus 函数，自动计算工作副本的状态：

  • new：新增的实体
  • dirty：有修改的实体
  • saved：未修改的实体

✅ 实时状态更新

所有工作副本更新函数现在都会：

  • 自动重新计算状态
  • 输出详细的调试日志
  • 确保界面实时响应

✅ 重置逻辑修复

resetWorkingCopy 现在会：

  • 真正重置数据到原始状态
  • 正确计算重置后的状态
  • 保持新增实体的 'new' 状态

🧪 测试步骤

1. 测试新增实体

  1. 打开实体列表页面
  2. 点击"添加实体"按钮
  3. 观察控制台日志
  4. 尝试编辑实体ID和名称

预期效果：

  • 输入框保持焦点
  • 状态显示为 'new'
  • 实时响应输入

2. 测试添加属性

  1. 在实体行点击"+"按钮添加属性
  2. 观察控制台日志
  3. 编辑属性ID和名称
  4. 修改属性类型

预期效果：

  • 属性立即出现在表格中
  • 输入框可以正常编辑
  • 状态自动变为 'dirty'

3. 测试现有实体编辑

  1. 编辑现有实体的ID或名称
  2. 观察状态变化
  3. 点击重置按钮
  4. 确认数据恢复原状

预期效果：

  • 修改时状态变为 'dirty'
  • 重置后状态变为 'saved'
  • 数据正确恢复

🚨 如果仍有问题

  • 工作副本状态是否正确计算
